Design Patterns#

If you are currently working on a big programming project, and you feel that your code is insufficient, not reusable, or not readable, or you have the pain to add new features, it’s probably the time to take a break and learn about design patterns.

Design patterns will help you to write your code in a scalable and readable way and will make your software better, sometimes a small design choice in your code would save you hours or days of work.

If you want to learn seriously about design patterns you can read a book called “head first design patterns”, this book explains design patterns in a simple and funny way. You can also read my blogs about some design patterns :)

Decorator pattern#

Scenario#

We have a class called ReadFile this class can take a file and read its content character by character, we want to add feature to our code so that we can transform characters to lowercase while reading these characters, We have the following code

public interface Read{
    public int read();
}

public class ReadFile implements Read{
    // This class implements the Read interface
    
    public ReadFile(String path){
        // code ...
    } 

    // this method will read a character each time and returns it 
    public int read(){
        // code ...
    }
}

We want to find a way to transform characters to lowercase while reading them but we don’t want to change anything in the ReadFile class, in this case, we can use the decorator pattern as follows

public class LowercaseFileReader implements Reader{
    
    private reader;

    public LowercaseFileReader(Reader reader){
        this.reader = reader;
    } 
}

first, we will create a decorator class, a decorator class will take a decorated object of type Reader and will also implements the interface Reader, so now we have to implement the read method on the decorator also and by this when we call the read method on the decorator it will redirect the call to the decorated object and it can also add behavior to this function

public int read(){

    // notice we call the method on the decorated object
    int c = reader.read();

    // now we return the lowercase version of c or -1 if there are no more characters
    return c == -1 ? c : Character.toLowerCase((char) c)
}

the magic of the decorator pattern is that the decorator object will take an object and decorate it with an additional behavior without changing anything in the decorated class and the decorator has the same type of the decorated class

Decorator pattern defined#

The decorator pattern attaches additional responsibilities to an object dynamically
